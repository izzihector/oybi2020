PERFORM POSTGRE DATA MANAGEMENT CONSULTATIONS IN ODOO

Run DML (Data manipulation language) queries from odoo directly to the database, without having to enter an administrative graphical interface such as pgAdmin III for the administration of PostgreSQL databases.

Save time with an easy to use interface

The interface will allow you to execute sql queries from a single view without the need for previous configurations.

Features

Create any type of data manipulation query and observe the results in the view.

Assign your users one or more permissions so they can run different queries:Read, Select, Insert, Update, Delete.

Keep a record of the SQL statements executed by the members of your work team.

Follow your team members and send messages, emails, inquiries, plan activities and create notes easily and all from the same module.

The module is designed to execute DML queries (data manipulation language: SELECT, INSERT, UPDATE, DELETE). If you want to execute the DDL (data definition language: CREATE, ALTER, DROP) and DCL (data control language: GRANT, REVOKE) commands, contact us to make the necessary extensions to the module.

Queries executed saved

SQL Query Control Save the executed queries, if you need to frequently execute one of these SQL instructions, you can modify it and execute it when you need it. These modifications do not alter the history of queries executed by the user, when modifying and executing a query, new queries executed in the history will be added.

Permission types

You can assign the permissions that each user will have to execute different types of queries in the database.

Permissions Feature:

Read: Read-only permission to observe the results of the query.

Select: Gives access to the selection instructions to make selection queries.

Insert: Gives access to the insert instruction to insert records in the database.

Update: Gives access to the update instruction

Delete: Gives access to the deletion declaration to delete records.

Activity Tracking

Analyze the queries executed by your work team, these will be stored in a history of executed instructions.
